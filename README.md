# Orbit — P2P мессенджер без сервера

## Как это работает
- Все данные (профиль, друзья, история чатов) хранятся **только в localStorage вашего браузера**.
- Сообщения, файлы, голосовые и звонки передаются **напрямую** между устройствами через WebRTC.
- Единственное, что приходит "извне" — это бесплатный публичный signaling-сервер библиотеки **PeerJS**, который нужен только на секунду, чтобы два браузера обменялись адресами и установили прямое соединение. Никакой Firebase, никакой БД, никакого хранения сообщений на сервере.
- **Важно:** сообщение доставится, только если получатель в этот момент онлайн (открыт сайт/приложение). Офлайн-очереди сообщений быть не может — их негде хранить без сервера.

## Деплой на GitHub Pages
1. Создайте новый репозиторий на https://github.com/new (можно публичный, приложение не хранит секретов)
2. Загрузите туда все файлы из этой папки (index.html, style.css, app.js, manifest.json, sw.js, icons/) — либо через веб-интерфейс "Add file → Upload files", либо через git:
   ```
   git init
   git add .
   git commit -m "Orbit messenger"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
   git push -u origin main
   ```
3. В репозитории: Settings → Pages → Source → выберите ветку `main` и папку `/ (root)` → Save
4. Через 1-2 минуты сайт появится по адресу `https://ВАШ_ЛОГИН.github.io/ВАШ_РЕПОЗИТОРИЙ/`
5. Откройте эту ссылку на двух устройствах — можно добавлять друг друга по ID

**Важно про обновления:** GitHub Pages отдаёт файлы через CDN (Fastly), который может кэшировать их на несколько минут. Если после обновления репозитория изменения не видны — подождите пару минут и сделайте жёсткое обновление страницы (на телефоне: закрыть вкладку/приложение полностью и открыть заново; в Chrome/Safari можно также через настройки очистить данные сайта). Сам Service Worker в этом проекте больше **ничего не кэширует** — он подключён только для того, чтобы браузер разрешил "Установить как приложение", все файлы каждый раз загружаются заново из сети.

## Локальная проверка перед деплоем
Нужен любой статический http-сервер (не открывайте index.html напрямую через file:// — service worker и getUserMedia требуют http/https):
```
python3 -m http.server 8080
```
затем откройте http://localhost:8080

## Установка как приложение (PWA)
На телефоне/десктопе в браузере после открытия сайта появится опция "Установить приложение" / "Добавить на экран Домой" — работает на Android, iOS (Safari → Поделиться → На экран «Домой»), Windows, macOS.

## Ограничения, о которых стоит знать
- Без сервера нет "офлайн-доставки" — оба должны быть в сети одновременно (или получатель должен зайти, пока отправитель ещё онлайн и идёт попытка переподключения).
- История чатов хранится в localStorage браузера — при очистке данных сайта или смене устройства она потеряется.
- Голосовые/фото/файлы кодируются в base64 и хранятся локально — для очень больших файлов (десятки МБ) браузер может отказать из-за лимита localStorage (~5-10МБ). Ограничение в приложении — 15МБ на файл.
- Для звонков и микрофона сайт должен быть открыт по HTTPS (Netlify это обеспечивает автоматически).
